# M2 — Multi-Source, Accounts, Private Beta (Design)

**Дата:** 2026-04-21
**Статус:** Draft → User Review
**Продолжает:** `2026-04-20-hotel-deals-bot-team-design.md` (top-level M0-M6 roadmap), `2026-04-21-m1-poc-design.md` (M1 PoC live на 193.233.208.24)

## Цели и scope

**Что M2 добавляет к уже живой M1:**

| Капзона | M1 (есть) | M2 (добавляем) |
|--|--|--|
| Источники | Booking.com (live + fixture fallback) | +Agoda, Hotels.com, Expedia, Ostrovok → **5 total** |
| Дубликаты | Один отель = один оффер | Merge: один отель = master + N предложений от разных источников |
| Юзеры | Анонимные session_id (Redis 2h TTL) | **Аккаунты** (email+password+verification) + invite-code гейт |
| История | Терялась при обновлении | **PG-персистентный ChatThread**, anon→logged-in миграция |
| Сохранение | — | **Saved hotels** с daily price refresh cron |
| Analytics | Только access logs | **Custom event_log + Grafana funnel dashboards** |
| Админка | Прямой SQL | **/admin/invites + /admin/beta** |

**Включено:**
- 4 новых Python-парсера по образцу Booking (Playwright + stealth + fixture fallback + geo-enrichment)
- Параллельный `scraper.search_all()` orchestrator + unified `HotelSource` Protocol
- Hotel dedup через геокоординаты (строго, 50m радиус, Postgres b-tree bbox prefilter)
- Новая master-модель `Hotel` + `OfferSnapshot.hotelId` FK
- Auth: email + password (bcrypt cost 12), email verification через Resend, JWT RS256 (1h access + 30d refresh rotation)
- Invite-only регистрация (уникальные коды в DB, TX redemption)
- Saved hotels + daily price refresh cron → detect ≥5% drops → queue для email в M2.5
- Persistent ChatThread + ChatMessage (PG), anonymous→logged-in claim workflow
- Custom event_log: 13 event types с fire-and-forget emitters + Grafana dashboards
- Admin UI для генерации invite кодов и просмотра беты

**Намеренно НЕ включено (→ M3 / M2.5):**
- GDPR полный (cookie consent, DSR endpoints) — отложен в M3, для приватной беты минимально
- Социальная регистрация (Google OAuth, magic link) — решение «email+password» в брейнсторм
- Email alerts о price drops — детекция есть, отправка писем в M2.5
- Публичные URL для саved hotels (shareable links)
- Мобильное приложение
- Прямые партнёрские API (переход в M3)
- PostHog / Mixpanel — используем custom event_log + Grafana

## Решения брейнсторм-сессии

| Вопрос | Выбор | Рационал |
|--|--|--|
| 4 новых источника | Agoda + Hotels + Expedia + Ostrovok | Глобальный mix + RU-рынок покрыт Ostrovok'ом |
| Auth метод | Email + password + verification | Классика, легко тестить, низкие риски для беты |
| GDPR | Отложить в M3 | Private beta = low risk; public launch в M3 получит полный compliance |
| Analytics | Custom (event_log + Grafana) | Полный контроль, данные под нашим PG, нет vendor lock-in |
| Dedup | Геокоординаты строго (50m) | Дёшево, точно, мало false merges; альтернативы (Google Place ID) — платно |
| Beta gate | Invite codes | Контроль списка, легко атрибутировать источник трафика |

## Секция 1 — Scraping (4 новых парсера + platform)

### Новые парсеры в `services/scraper/src/scraper/`

| Файл | Источник | Приоритетные регионы |
|--|--|--|
| `agoda.py` | Agoda | Asia (Thailand, Japan, Indonesia) |
| `hotels_com.py` | Hotels.com | US, UK, Australia |
| `expedia.py` | Expedia | Global (bundles) |
| `ostrovok.py` | Ostrovok | RU, СНГ, есть RUB-прайсинг на выдаче |

Каждый парсер реализует `HotelSource` Protocol:

```python
# services/scraper/src/scraper/sources.py
from typing import Protocol
from .types import SearchResult, HotelDetails

class HotelSource(Protocol):
    name: str                 # "booking.com", "agoda.com", ...
    currency: str             # default source currency
    rate_limit_rps: float     # requests per second ceiling

    async def search(
        self,
        destination: str,
        checkin: date,
        checkout: date,
        adults: int = 2,
        children: int = 0,
        **filters,
    ) -> SearchResult: ...

    async def get_details(self, booking_url: str, offer_id: str) -> HotelDetails: ...

    async def health(self) -> dict: ...  # {"status": "ok" | "degraded" | "down", "latency_ms": int, "blocked_rate_pct": float}
```

Реализация парсера = Playwright + stealth + DC-прокси + fallback на фикстуры + geo-extract. Примерный размер per parser: ~300 LOC + 5 fixture HTML files + 5 regression tests.

### Registry + параллельный search_all

```python
# services/scraper/src/scraper/registry.py
SOURCES: dict[str, HotelSource] = {
    "booking.com":   BookingSource(),
    "agoda.com":     AgodaSource(),
    "hotels.com":    HotelsComSource(),
    "expedia.com":   ExpediaSource(),
    "ostrovok.ru":   OstrovokSource(),
}

async def search_all(
    destination: str, checkin: date, checkout: date,
    adults: int = 2, children: int = 0,
    sources: list[str] | None = None,   # None = все
    **filters,
) -> dict[str, SearchResult]:
    selected = [SOURCES[s] for s in (sources or list(SOURCES.keys()))]
    tasks = [
        s.search(destination, checkin, checkout, adults, children, **filters)
        for s in selected
    ]
    results = await asyncio.gather(*tasks, return_exceptions=True)
    return {
        selected[i].name: (r if not isinstance(r, Exception) else _error_result(r))
        for i, r in enumerate(results)
    }
```

### Geo extraction

Каждый парсер обязан пытаться извлечь `lat, lng` из:
1. JSON-LD `<script type="application/ld+json">` с `Hotel.geo.latitude/.longitude` (большинство сайтов)
2. `data-lat / data-lng` атрибуты на карточке (некоторые источники)
3. Map-script parsing (крайний случай, например Booking.com хранит в window-var)

Если координаты не найдены → оффер сохраняется с `lat=null, lng=null` → dedup не работает → `hotel_id=null` в БД → оффер показывается как orphan в UI.

Метрика: % offers with geo per source. Если <80% → scraping-engineer обновляет селектор.

### Rate limiting + proxy budget

- `asyncio.Semaphore(2)` per domain в registry (гарантирует max 2 concurrent reqs на booking.com одновременно)
- 3-5 сек рандомизированная пауза между последовательными запросами
- Прокси-пул shared между источниками (из `PROXY_URLS` env)

### Fixture pool

Добавляем 4 фикстурных города per source: Barcelona, Bangkok, Moscow, NYC → 16 новых HTML файлов (~1-2MB каждый) + 20 regression тестов (`≥15 offers parsed with valid fields`).

### Health monitoring

- Каждый парсер при каждом вызове пишет event `scraper.source_result` с `source, offers_count, blocked_bool, latency_ms`
- Grafana dashboard «Scraper Health» показывает success rate per source за 24h
- Алерт: если <70% за 15 мин → Grafana sends email/webhook (настроено в M3)

### Matching алгоритм (в ai-agent, async)

```python
async def match_or_create_hotel(
    pool: asyncpg.Pool,
    offer: HotelOffer,
) -> str | None:
    if offer.lat is None or offer.lng is None:
        return None

    # Bbox prefilter (~55m по широте на экваторе) для скорости
    BBOX = 0.0005
    async with pool.acquire() as conn:
        row = await conn.fetchrow("""
            SELECT id FROM hotel
            WHERE lat BETWEEN $1 - $3 AND $1 + $3
              AND lng BETWEEN $2 - $3 AND $2 + $3
              AND (
                  6371000 * acos(
                      cos(radians($1)) * cos(radians(lat)) *
                      cos(radians(lng) - radians($2)) +
                      sin(radians($1)) * sin(radians(lat))
                  )
              ) <= 50
            ORDER BY id
            LIMIT 1
        """, float(offer.lat), float(offer.lng), BBOX)

        if row:
            return row["id"]

        hotel_id = await conn.fetchval("""
            INSERT INTO hotel (id, "canonicalName", lat, lng, stars, city, "createdAt")
            VALUES ($1, $2, $3, $4, $5, $6, now())
            RETURNING id
        """, cuid(), offer.hotel_name, float(offer.lat), float(offer.lng), offer.stars, offer.city)
        return hotel_id
```

Важно: операция не атомарная между SELECT/INSERT → возможны дубли при гонках. Приемлемо для M2 (коррекция раз в неделю через merge-cron в M2.5). Альтернатива — `SELECT ... FOR UPDATE` + INSERT в транзакции, но добавляет overhead.

### Security

- **Пароли:** bcrypt cost=12 (~300ms hash time). Min 8 chars (никаких других ограничений).
- **JWT:**
  - RS256, keys в `JWT_PRIVATE_KEY_BASE64` / `JWT_PUBLIC_KEY_BASE64` env (генерируются `openssl genpkey -algorithm RSA -pkeyopt rsa_keygen_bits:2048`).
  - Access TTL 1h. Payload `{sub:userId, email, iat, exp, sid}`.
  - Refresh TTL 30d, opaque random 64 bytes, rotated на каждом `/auth/refresh` (старый немедленно инвалидируется).
- **Rate limiting** (`@nestjs/throttler`):
  - `/auth/login` — 5/min per IP
  - `/auth/signup` — 3/min per IP
  - `/auth/request-password-reset` — 2/min per email
- **Invite redemption — транзакционный:**
  ```sql
  BEGIN;
  SELECT * FROM invite_code WHERE code = $1 AND "redeemedBy" IS NULL
    AND ("expiresAt" IS NULL OR "expiresAt" > now()) FOR UPDATE;
  -- if row → INSERT user + UPDATE invite_code SET redeemedBy=..., redeemedAt=now()
  COMMIT;
  ```

### ChatController.turn() расширение

Поведение в M2:
- Если запрос авторизован → используем/создаём `ChatThread` (PG), пишем `ChatMessage` записи для user+assistant.

### Price refresh cron

Отдельный service `apps/api/src/cron/refresh-saved-prices.ts`:

```typescript
// Runs daily 03:00 UTC via systemd timer
async function run() {
  const saved = await prisma.savedHotel.findMany({
    where: { hotel: { lat: { not: null } } },
    include: { hotel: true },
    take: 500,  // cap: 500 hotels / day
  });

  for (const batch of chunk(saved, 10)) {
    await Promise.all(batch.map(async (s) => {
      const today = new Date();
      const in7 = new Date(Date.now() + 7 * 864e5);
      const result = await fetchAiAgentSearchAll({
        destination: s.hotel.city, checkin: today, checkout: in7,
      });

      // Find offers for this master hotel
      const offers = result.master_hotels.find(h => h.hotel_id === s.hotelId)?.offers ?? [];
      if (offers.length === 0) return;

      const newMin = Math.min(...offers.map(o => o.price_per_night_usd));
      const previousMin = await getLatestSnapshotPrice(s.hotelId);

      if (previousMin && newMin < previousMin * 0.95) {
        const dropPct = Math.round((1 - newMin / previousMin) * 100);
        await prisma.priceAlertQueue.create({
          data: { userId: s.userId, hotelId: s.hotelId, oldPriceUsd: previousMin, newPriceUsd: newMin, dropPct },
        });
      }
    }));
    await sleep(1000);  // throttle across batches
  }
}
```

Запуск: `hoteldeals-price-cron.service` + `hoteldeals-price-cron.timer` (OnCalendar=daily).

Бюджет proxy: 500 saved hotels × 5 sources = 2500 scraper calls/day. 2500 × $0.003/call (Webshare) ≈ $0.75/day = $23/mo. Acceptable.

### Emitters

**TS (apps/api + apps/web):**

```typescript
// packages/analytics-client/src/emit.ts
let buffer: EventRow[] = [];
let flushTimer: NodeJS.Timeout | null = null;

export function emit<N extends EventName>(name: N, props: EventProps[N], ctx?: { userId?: string; threadId?: string; sessionId?: string; source: Source }) {
  buffer.push({ name, props, ts: new Date(), ...ctx });
  if (!flushTimer) {
    flushTimer = setTimeout(flush, 200);
  }
  if (buffer.length >= 100) flush();
}

async function flush() {
  if (flushTimer) { clearTimeout(flushTimer); flushTimer = null; }
  if (buffer.length === 0) return;
  const toSend = buffer; buffer = [];
  try {
    await prisma.eventLog.createMany({ data: toSend, skipDuplicates: false });
  } catch {
    // fire-and-forget — не блокируем бизнес-логику
  }
}
```

**Python (ai-agent, scraper):**

```python
# services/shared-events/src/events.py (симметрично)
import asyncpg, asyncio, json

_buffer: list[dict] = []
_flush_lock = asyncio.Lock()

async def emit(pool, name: str, props: dict, ctx: dict | None = None):
    _buffer.append({"name": name, "props": json.dumps(props), **(ctx or {})})
    if len(_buffer) >= 50:
        asyncio.create_task(_flush(pool))

async def _flush(pool):
    async with _flush_lock:
        if not _buffer: return
        rows = list(_buffer); _buffer.clear()
    try:
        async with pool.acquire() as conn:
            await conn.executemany(
                'INSERT INTO event_log (name, "userId", "threadId", "sessionId", source, props) VALUES ($1,$2,$3,$4,$5,$6)',
                [(r["name"], r.get("userId"), r.get("threadId"), r.get("sessionId"), r.get("source","ai-agent"), r["props"]) for r in rows]
            )
    except Exception:
        pass  # fire-and-forget
```

## Риски и митигации

| Риск | Митигация |
|--|--|
| 4 новых сайта блокируют скрейпинг → много orphan offers | Fixture fallback per source; Playwright stealth; план перехода на Affiliate APIs в M3 |
| Dedup false positives (один отель в 2 master записях из-за гонки) | Bbox + дальнейший merge-cron раз в неделю в M2.5 |
| Resend email попадает в спам | SPF/DKIM, text+HTML, unsubscribe link |
| Price cron грузит сайты → global ban | Throttle 500 hotels/day, distributed по 24h, resident proxies фолбэк |
| Invite code двойная редемпция | Транзакционный SELECT FOR UPDATE + UPDATE |
| Anonymous→logged-in claim: чужие session_ids | LocalStorage ∈ client, cap ≤5 threads, `POST /claim` no-op для чужих тредов (WHERE userId IS NULL) |
| JWT key compromise | Key rotation: add `kid` header, поддерживаем 2 активных ключа в M3 |
| Grafana public exposure | Только 127.0.0.1 binding, access через SSH tunnel, публичный доступ в M3 |

## Open questions (для ADR на этапе реализации)

- **Domain & DNS:** купить `hoteldeals.app` или другой? DNS provider — Cloudflare (рекомендую) или Namecheap?
- **Resend vs SMTP self-hosted:** Resend за $20/mo после free tier; self-hosted MTA — сложно для beta, рекомендую Resend.
- **bcrypt cost=12 vs argon2id:** argon2id современнее и устойчивее к GPU attacks, но добавляет dep `@node-rs/argon2`. Разница в security minimal для 50-user беты. bcrypt проще.
- **RS256 vs ES256:** RS256 более распространён и отлаживается в JWT debugger'ах. ES256 меньше payload (короче токены). Для нашего случая — RS256.
